  • 如果ping显示没有网络，那就先ifconfig；如果只显示lo，没有网卡，如下图

  • 查看所有的网络设备，输入ifconfig -a命令

  • 启动网卡，输入ifconfig xxx up；xxx是自己的网络设备名字，如ens33
  • 之后开始设置IP地址，使用dhclient工具自动设置IP地址

  • 这里的地址192.168.219.130是动态的，下次就会变化；接下来设置配置文件，将IP地址设置成静态！(因为远程连接的话，每次IP地址都会变，不适合学习)

局域网内使用

  • 上面的代码成功后，接下来就是如何优化环境了，这部分给出参考链接，从 局域网内使用 开始看

    • 直接在上面的例子中添加其余 IP 地址即可
    • mkcert localhost 127.0.0.1 ::1 192.168.xxx.xxx 顺序影响的只是文件名，有几个地址就会出现 + n 的字样
    • 这里重新生成的时候，注意测试样例的密钥文件也要变

  • 接下来分发证书并信任，然后通过对应的地址访问（由于端口问题，谷歌和 Edge 暂时还访问不了，用 IE 可以）

    • 进入 ~.local/share/mkcert/ 目录，将 rootCA.pem 拷贝一个副本 rootCA.crt
    • rootCA.crt 是分发的关键，下面写三个系统信任的方法

  • Win 10

    • 直接托文件，然后双击，要将证书存储为 受信任的根证书颁发机构

  • Ubuntu

    • 进入 rootCA.crt 所在的目录，然后拷贝 sudo cp rootCA.crt /usr/local/share/ca-certificates/
    • 更新并信任 sudo update-ca-certificates
    • Ubuntu 下可以使用 curl -I https://192.168.xxx.xxx 来测试

  • CentOS

    • sudo cp rootCA.crt /etc/pki/ca-trust/source/anchors/
    • sudo update-ca-trust

  • 再记录下防火墙怎么查看状态开启关闭

    • Ubuntu
      • sudo ufw status、sudo ufw enable、sudo ufw disable
    • CentOS
      • systemctl status/start/enable/stop/disable firewalld.service

小结

  • 其实按照上面步骤，我的电脑是没有问题的！不过经手我朋友（欣）的笔记本后，我发现了新天地！

    • 先配置静态 IP 地址，之后生成证书，服务器本地是可以访问的，分发给 Win10 不可信，分发给其余节点不可信……原因不明！
    • 思索无果后，下午欣带回去准备重新生成一遍，之后主机无法访问虚拟机，但是虚拟机可以访问主机……原因不明！
    • 晚上开始重点解决这个问题，半天没头绪后重启，居然又可以连了……为防止其他意外，一切重头开始！

  • 之后几经曲折，最终成功，我悟出了几个道理

    • 重启可以解决 90% 的问题
    • CA 证书生成一次即可，终生受用；需要改变的是服务端程序要用到的两个密钥文件
    • 至于证书不可信，那是因为还没有添加信任，直接在浏览器里（什么谷歌、火狐、Edge 等等）添加证书，命令行不顶用，为什么？别问我，因为我不知道
    • 之后添加域名，其实就是 IP 地址的别名，由于是自定义的，与网络上已有的无关（或者网络上没有），那么就需要添加 host 文件了，将服务器地址与域名绑定即可；注意任意需要通过域名来访问服务端都需要添加 host 文件
