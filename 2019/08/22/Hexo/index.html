Git

本地操作

  • 先设置本地的用户签名，可以是虚拟的，只是方便标识提交者
  • 初始化本地库，这里需要新建文件夹 git init
  • 查看本地库状态，是否有改动提交等 git status
  • 添加到暂存区即 .git 隐藏文件夹 git add yourFile or git add .
  • 提交到本地库 git commit -m "yourInfo" yourFile
  • 查看历史记录 git reflog or git log
  • 指定版本即移动 HEAD 指针 git reset --hard versionNumber

分支

  • 创建分支 git branch branchName
  • 查看分支 git branch -v
  • 切换分支 git checkout branchName
  • 将指定分支合并到当前分支上 git merge branchName
    • 合并的时候，对应文件一般只有一方修改，若双方都修改了同一文件的不同地方，需手动合并。

解决冲突

  • 修改代码，会有符号提示 MERGING
    • <<<<<<<HEAD xxxCode ======= yyyCode >>>>>>>hot-fix
    • 手动删除不需要的代码
  • 添加到暂存区 git add yourFile
  • 提交，无需文件名 git commit -m "merge hot-fix"
  • MERGING 消失

远程仓库

  • 查看当前所有远程地址别名 git remote -v
  • 添加远程地址并起别名一般都是 origin git remote add aliasName yourAddress
  • 推送本地分支上的内容到远程仓库 git push aliasName yourBranch
  • 下载 git clone remoteAddress
  • 拉取远程仓库对应分支的最新内容并与本地分支直接合并 git pull aliasName branchName

SSH

  • 之前的地址是 HTTPS 的需要登录，而 SSH 不需要
  • 进入当前用户的主目录，随便哪个目录也行，生成密钥 ssh-keygen -t rsa -C "youremail@example.com"
    • 生成的密钥在 C 盘用户的主目录下
  • 将 id_rsa.pub 添加到 GitHub 即可

Backup

  • 新建一个仓库，将原来的备份仓库删除，新建的时候有提示如何备份
    • 如果上传找不到仓库，则没有建立连接
    • 如果提示已存在仓库，则运行 rm origin 再重新建立连接

若是遇到其他问题，根据提示来即可。参考链接

Recover

下载

  • 下载仓库，本地 git clone 地址 （先添加 SSH 公钥）
  • 下载 node.js 并且勾选添加环境变量

node.js

环境

  • 查看 git 版本 git --version
  • 查看 node 版本 node -v
  • 查看 npm 插件版本 npm -v
  • 安装 Hexo，npm install hexo-cli -g 默认安装到 C 盘
  • 新建文件夹，并运行 hexo init 不要在备份的文件夹中使用
    • 替换默认的 _config.yml、themes/、source/、scaffolds/、packagejson、.gitignore
  • 查看 hexo 版本 hexo -v
    • 运行 hexo g、hexo s 是否成功，如不成功则根据提示来安装 node.js node_modules
  • 之前因为安装了几个插件，所以 hexo s 有点问题
    • 加密插件 npm install --save hexo-blog-encrypt 就不安装了，配置文件也恢复了
    • 置顶插件 npm install hexo-generator-topindex --save 卸载 npm uninstall hexo-generator-topindex
  • 总的来说，这里写的有点旧了，好多插件都更新会出现新的问题，需要跟上潮流、随机应变！

Deployment

部署 GitHub

  • 插件：使用 hexo d 来部署到 git 上，需要安装 npm install hexo-deployer-git --save
  • 部署的时候会出现 node 版本过高的情况而无法部署，解决方法主要有两种，降低 node 版本以及使用 nvm 来控制 node 版本，这里选择第二种来记录下
    • 点击安装版本，会分别安装 nvm 以及 nodejs symbolic link （文件夹内不显示）不要出现空格
    • nvm -v、nvm ls、nvm list available、nvm install x.x.x
    • 用管理员身份运行 nvm use x.x.x
    • 卸载 nvm uninstall x.x.x
    • node -v
  • 还有个问题：npm 无法运行
    • npm 高于 8.11 版本要下载安装包解压 https://nodejs.org/download/release/vx.x.x/
    • 解压目录 nvm 安装路径，并将解压的文件夹命名 vx.x.x 格式
    • 之后再用管理员打开 cmd 在重新 use 下版本 node -v、npm -v
