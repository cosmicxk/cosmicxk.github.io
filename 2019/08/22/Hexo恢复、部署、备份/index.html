Hexo恢复、部署、备份

发表于 2019-08-22 更新于 2020-11-12 分类于 部署 阅读次数： Valine：

恢复

下载

  • 复制之前备份的仓库地址

  • 本地git clone 地址

    git clone 地址

环境

  • git下载安装一路next(git-for-windows)

  • 查看git版本git --version

  • node.js安装13或者12版本时一路next，勾选添加环境变量(node.js下载)

  • 查看node版本node -v

  • 查看npm插件版本npm -v

    查看版本用cmd或者git都可

  • 安装Hexo，(Hexo文档)运行npm install hexo-cli -g默认安装到C盘；如果是直接安装的话，运行hexo init 文件名(可有可无)会生成新的文件，不要在备份的文件夹中使用否则_config.yml,theme/,source/,scaffolds/, packagejson, .gitignore,会变成默认值。

  • 查看hexo版本hexo -v；到了这一步，就可以在备份的文件夹中运行hexo g
    、hexo s是否成功，如不成功则根据提示来(安装node.js node_modules)。http://localhost:4000

    至此本地恢复已经完成

部署

插件

  • 使用hexo d来部署到git上,需要安装npm install hexo-deployer-git --save

密钥

  • 在备份的文件夹里运行ssh-keygen -t rsa -C "youremail@example.com”
  • 在C盘找到rsa.pub，打开复制里面的字符串添加到github上；应该有三个文件，如果只有两个，再运行一次ssh-keygen -t rsa -C "youremail@example.com”，之后输入yes并且回车。
  • 验证是否链接成功，运行ssh -T git@github.com

不要在git版本库中运行ssh,然后又将它提交，这样就把密码泄露出去了。

  • 设置用户名、邮箱用以提交

部署

  • github中新建一个仓库用户名+github.io，并复制链接；可以将之前的删除(看来必须删，不然部署不了)，重新创建一个。
  • 修改配置文件_config.yml
  • 部署三步走

访问用户名.github.io即可，我之前是绑定了域名，后来域名过期了，要解绑，先在域名解析那儿解绑；再将soure里的CNAME文件删除即可。

备份

  • 新建一个仓库，将原来的备份仓库删除，新建的时候有提示如何备份
  • 如果上传找不到仓库，则没有建立连接
  • 如果提示已存在仓库，则运行rm origin，再重新建立连接

  • 文本编辑 git commit
  • 按键盘字母 i 进入insert模式
  • 按键盘左上角”Esc”
  • 输入”:wq”,注意是冒号+wq,按回车键即可

若是遇到其他问题，根据提示来即可。参考链接
https://blog.csdn.net/qq_29525875/article/details/96598237
