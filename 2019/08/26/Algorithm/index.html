快速排序

速度快、效率高！

  • 分治法；将一个数组分为两个部分，并且选择一个数字作为基数
  • 菜鸟网站的代码看的有点蒙，先看书上怎么写的
    • 一个是主函数QuickSort、一个是分区函数Paritition
    • 主函数里面是将分区函数返回的一个索引（位置），来分成两个数组（不包括索引本身！），然后两个数组递归！
    • 排序的主体在分区函数，所以主要看分区函数，是如何将小的大的分别放两边的！
      • Paritition函数参数——数组（指针）、数组长度、起始点、结束点
      • 判断是否为空、长度为负、两个标记点在不在数组边界内
      • 书上还用了随机数函数，用来在两个标记点之间取随机数，力扣上没有随机函数
      • 有个问题，int small = start - 1;为什么要减一呢，如果为0，岂不是-1……没事了
      • 先把基数与最后一位交换，之后遍历到倒数第二位，遍历的时候用small来记录最后一个小于基数的位置，初始值为start - 1；如果这个数比基数（end所在的数）大，什么也不干，下一个；如果这个数小，++small；，如果small与遍历的位置相等，说明前面没有大的数字，若不等，说明small指的是大的数，与小的数交换，之后下一个
    • 总结，small在交换之后，总是指向小的数字，遍历结束之后，要++small与基数交换，最后返回small，也就是基数的位置

  • 冒泡排序类似，每次把小的放前面…
  • 力扣写的似乎更牛批点
    • 基数就是int pivot = A[low]，然后右标记点左移，直到小于pivot
    • 此时A[high]是小数，然后交换，或者赋值A[low] = A[high];
    • 之后左标记点右移，直到大于pivot，然后赋值A[high] = A[low];
    • 一直赋值，最终A[low] = pivot;，返回low，代码就不放了

堆排序

  • 大根堆：每个节点的值都大于或等于其子节点的值，在堆排序算法中用于升序排列；

  • 小根堆：每个节点的值都小于或等于其子节点的值，在堆排序算法中用于降序排列；

  • 实质是给数组排序

    • 建堆和交换得好好理解下
    • 堆化的顺序得从下往上（就是交换节点，确保父节点值大于等于子节点值）
    • 而且容易错，对于大根堆，如果根小于叶子，必须逐级递归调整才行

  • 注意标号

    • 完全二叉树是从上往下、从左往右；就从 0 开始吧，根数组下标对应起来
    • 对于某个标号 i 也就是数组里的下标
      • 该左节点的标号为 2 * i + 1
      • 该右节点的标号为 2 * i + 2
      • 同样，给出下标 i 的节点，其父节点为 (i - 1) / 2

  • 一般来说从堆顶开始堆化，不能确保最后的顺序是正确的

    • 而是要从最后一个非叶节点倒序进行堆化（这是一个完全二叉树）
    • 正确来说是最后一个节点的父节点开始！

  • 堆化确保所有父节点的值大于等于子节点值即可，而不是严格降序

    • 之后排序，就是把最大值和最后一个节点进行交换，然后隔开最后一个节点（个数减一即可），再进行堆化

背包问题

  • 这个应该放在动态规划下面，不过动态规划还没有整理，就先这样吧！
  • 关键词——重量、价值
  • 以下三类区别——（不一定满，价值最大）、（恰好装满，能或不能）、（恰好装满，一共多少种类）

0-1背包问题

  • 要么装进背包里，要么不装——非0即1
  • 状态
    • 状态有两个，就是「背包的容量」和「可选择的物品」
  • 选择
    • 「装进背包」或者「不装进背包」
  • 套框架

  • 明确dp数组的定义
    • 状态有两个，所以是两维——一维表示可选择的物品，一维表示背包的容量。
    • dp[i][w]的定义如下：对于前i个物品，当前背包的容量为w，这种情况下可以装的最大价值是dp[i][w]。
    • 这么定义，便于状态转移。别问，问就是不会！
    • 最终答案就是最后一行最后一列的数

  • 之后关键点在于，选择了这个物体，转移方程是什么，对应的重量要减去这个物体——即在装第i个物品的前提下，背包能装的最大价值是多少

注意，这里背包的容量不一定是满的，而且物品有限

子集背包问题

  • 这里就不是价值最大了，而是能不能！
  • 因为分为两个子集，要让这两个子集总和相等，那就背包容量就是总和的一半

  • 这里动态规划的矩阵，也就变成了布尔值！但是也有共性，dp[i][j]就是只对前i个数进行取值
  • 因为物品有限，这个数取完了，那么状态转移的时候，就变成了dp[i - 1][j-nums[i-1]]——如果物品无限，也就是完全背包，这个取走之后还有这个数，神不神奇！那就是dp[i][j-nums[i-1]]

  • 框架如下

  • 状态压缩！后序遍历是因为每个物品（数字）只能使用一次，以免之前的结果影响其他的结果——然而完全背包就不需要后序遍历了，因为物品无限，结果不影响

注意，这里背包的容量必须是满的，而且物品有限

完全背包问题

  • 区别是每个物品的数量无限；而且一般来讲，题目都会问给定容量（重量），然后恰好等于容量（重量）的物品种类！比如兑换硬币
  • 但是，要会举一反三
    • 比如279.平方数这题，这里就不是问多少种了，而是问最少的个数
    • 因此，状态转移也会变，不是直接相加了——而是比较哪个小，对应的dp[i][j]含义也不一样，这里是最小的个数
    • 最重要的就是边界，这决定了后面的状态！
